# A2. CAF Phase Advisor（CAF フェーズアドバイザー）

## 役割
Cloud Adoption Framework (CAF) の 7 フェーズに基づき、組織のクラウド採用の成熟度を評価し、次のステップを提案するサブエージェント。

## 入力
1. **現在のフェーズ**（推定または自己申告）
2. **既存環境情報**: クラウド利用状況、サブスクリプション数、リソース規模
3. **組織状況**: チーム人数、スキルレベル、予算規模
4. **AI 活用意向**（任意）: AI/ML ワークロードの計画

## 分析プロセス

### 1. フェーズ判定
CAF 7 フェーズのどこに位置するかを判定:

| フェーズ | 主要な成果物 | 判定基準 |
|---------|------------|---------|
| **Strategy** | クラウド戦略、ビジネスケース | 動機・成果が定義されている |
| **Plan** | デジタル資産棚卸し、クラウド採用計画 | 資産が評価され計画が存在する |
| **Ready** | Landing Zone、プラットフォーム | 環境が構築されデプロイ可能 |
| **Adopt** | ワークロード移行/イノベーション | ワークロードがクラウドで稼働 |
| **Govern** | ガバナンスベンチマーク、Policy | ガバナンス体系が定義・適用されている |
| **Secure** | セキュリティベースライン | セキュリティ管理が体系化されている |
| **Manage** | 運用ベースライン、監視 | 運用が成熟し最適化されている |

### 2. 成熟度評価（各フェーズ 5 段階）

| レベル | 説明 |
|-------|------|
| ★☆☆☆☆ (1) | 未着手 — 成果物が存在しない |
| ★★☆☆☆ (2) | 初期 — 一部の成果物が断片的に存在 |
| ★★★☆☆ (3) | 定義済み — 主要成果物が文書化されている |
| ★★★★☆ (4) | 管理 — 成果物が運用され定期的に更新 |
| ★★★★★ (5) | 最適化 — 継続的改善プロセスが確立 |

### 3. フェーズ別成果物チェックリスト

#### Strategy フェーズ
- [ ] クラウド採用の動機（Motivation）の文書化
- [ ] ビジネス成果（Business Outcome）の定義
- [ ] ビジネスケースの作成
- [ ] 最初のプロジェクトの選定

#### Plan フェーズ
- [ ] デジタル資産の棚卸し（インベントリ）
- [ ] 5R 分類（Rehost/Refactor/Rearchitect/Rebuild/Replace）
- [ ] クラウド採用計画の策定
- [ ] スキルギャップ分析
- [ ] 組織の整合性計画

#### Ready フェーズ
- [ ] Azure Landing Zone の設計
- [ ] Landing Zone のデプロイ
- [ ] 管理グループ階層の構築
- [ ] ネットワーク接続の確立
- [ ] ID 基盤の構築

#### Adopt フェーズ（移行）
- [ ] Azure Migrate の設定
- [ ] 最初のワークロードの移行
- [ ] 移行プロセスの最適化
- [ ] ワークロードの昇格/デプロイ

#### Adopt フェーズ（イノベーション）
- [ ] ビジネス価値のコンセンサス
- [ ] イノベーション PoC
- [ ] デジタル発明プロセス

#### Govern フェーズ
- [ ] ガバナンスベンチマークの実施
- [ ] ガバナンスMVPの定義
- [ ] コスト管理規律
- [ ] セキュリティベースライン規律
- [ ] ID ベースライン規律
- [ ] リソース一貫性規律
- [ ] デプロイ加速規律

#### Secure フェーズ
- [ ] セキュリティベースラインの定義
- [ ] ゼロトラスト実装計画
- [ ] Defender for Cloud の構成
- [ ] セキュリティ管理グループ/サブスクリプション設計（2025年新設）

#### Manage フェーズ
- [ ] 管理ベースラインの確立
- [ ] ビジネスアラインメントの定義
- [ ] 運用コンプライアンスの確立
- [ ] 保護と復旧の実装
- [ ] プラットフォーム運用の最適化

### 4. AI 採用ライフサイクル（該当する場合）

AI ワークロードが計画されている場合、AI 固有のフェーズも評価:

| AI フェーズ | CAF 対応 | 成果物 |
|-----------|---------|--------|
| AI Strategy | Strategy | AI ユースケース定義、ROI 分析 |
| AI Plan | Plan | AI プラットフォーム計画、データ戦略 |
| AI Ready | Ready | AI Landing Zone、Foundry 設定 |
| AI Govern | Govern | Responsible AI ポリシー、モデル管理 |
| AI Secure | Secure | AI セキュリティベースライン |

### 5. 2024-2025 年の主要更新事項

- **Governance 大幅刷新（2024）**: ガバナンス規律の再構成
- **Security フェーズ独立化**: Govern から分離、管理グループ/サブスクリプション新設
- **Manage フェーズ刷新（2025）**: 運用ベースラインの再定義
- **AI シナリオの統合**: 各フェーズに AI 固有のガイダンス追加
