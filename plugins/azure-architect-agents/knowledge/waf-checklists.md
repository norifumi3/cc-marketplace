# WAF チェックリスト — Azure Well-Architected Framework

## 概要
Azure Well-Architected Framework は 5 つの柱で構成され、各柱にチェックリスト項目がある。

---

## 1. Reliability（信頼性）

### チェックリスト項目

| ID | タイトル | 概要 |
|----|---------|------|
| **RE:01** | 信頼性の設計 | 信頼性要件を特定し、目標を設定する |
| **RE:02** | 復旧目標 | RPO/RTO を定義し、ワークロードの復旧を計画する |
| **RE:03** | 障害モード分析 | 潜在的な障害モードを特定し、影響を評価する |
| **RE:04** | 冗長性 | すべてのレイヤーで冗長性を設計する |
| **RE:05** | マルチリージョン | 高可用性のためマルチリージョン構成を検討する |
| **RE:06** | データ整合性 | データのレプリケーションと整合性を管理する |
| **RE:07** | ヘルスモデリング | コンポーネントとフローのヘルスを監視する |
| **RE:08** | 自己修復 | 自己修復メカニズムを実装する |
| **RE:09** | テスト | 障害テスト（Chaos Engineering）を実施する |
| **RE:10** | バックアップ | バックアップと復元の戦略を実装する |
| **RE:11** | DR 手順 | DR 手順を文書化しテストする |
| **RE:12** | スケーリング | 負荷変動に対応するスケーリングを設計する |
| **RE:13** | アップデート | 安全なアップデートとデプロイメントを設計する |

### 主要デザインパターン
- Circuit Breaker / Retry / Bulkhead / Queue-Based Load Leveling
- Health Endpoint Monitoring / Compensating Transaction
- Deployment Stamps / Geode

## 成熟度モデル（2025年追加）

各柱の成熟度を 5 段階で評価:

| レベル | 説明 |
|-------|------|
| 1. Initial | アドホック、ドキュメントなし |
| 2. Developing | 基本的なプラクティスが一部存在 |
| 3. Defined | プラクティスが文書化・標準化 |
| 4. Managed | メトリクスに基づく管理 |
| 5. Optimized | 継続的改善が自動化・体系化 |
