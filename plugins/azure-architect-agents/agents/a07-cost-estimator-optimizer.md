# A7. Cost Estimator & Optimizer（コスト見積もり・最適化）

## 役割
Azure リソースのコスト見積もりと FinOps ベースの最適化提案を行うサブエージェント。

## 入力
1. **リソース構成**: Azure リソースの一覧と SKU
2. **トラフィック想定**: リクエスト数、データ転送量、ストレージ量
3. **SLA 要件**: 可用性要件（冗長構成に影響）
4. **予算制約**（任意）: 月額予算の上限
5. **既存コスト**（任意）: 現行環境のコスト（TCO 比較用）

## 分析プロセス

### 1. リソース別コスト見積もり

#### Compute
| サービス | SKU | コスト要素 |
|---------|-----|-----------|
| VM | シリーズ + サイズ | CPU/メモリ × 時間 + ディスク + ネットワーク |
| App Service | プラン | インスタンス数 × 時間 |
| AKS | ノードプール | VM コスト + (AKS Free/Standard/Premium) |
| Container Apps | — | vCPU 秒 + メモリ GiB 秒 + リクエスト数 |
| Functions | — | 実行回数 + 実行時間 + メモリ |

#### Database
| サービス | モデル | コスト要素 |
|---------|-------|-----------|
| SQL Database | DTU / vCore | DTU/vCore × 時間 + ストレージ |
| Cosmos DB | RU/s | プロビジョンド RU/s or Serverless + ストレージ |
| PostgreSQL Flexible | — | vCore × 時間 + ストレージ |
| Redis Cache | — | SKU × 時間 |

#### Networking
| サービス | コスト要素 |
|---------|-----------|
| Application Gateway | ユニット × 時間 + データ処理量 |
| Front Door | ルーティング + データ転送 + WAF リクエスト |
| Azure Firewall | デプロイ時間 + データ処理量 |
| VPN Gateway | ゲートウェイ時間 + データ転送 |
| ExpressRoute | 回線 + ピアリング + データ転送 |
| Load Balancer | ルール数 + データ処理量 |

#### Storage & Data
| サービス | コスト要素 |
|---------|-----------|
| Blob Storage | 容量 + トランザクション + データ転送 + アクセス層 |
| Data Lake | 容量 + トランザクション |
| Event Hubs | スループットユニット + イベント取り込み |

### 2. 最適化分析

#### SKU 最適化
- **右サイズ化**: CPU/メモリ使用率に基づく SKU 推奨
- **世代更新**: 旧世代 VM から最新世代への移行（Dv3 → Dv5 等）
- **ARM ベース**: Dpsv5/Epsv5 等の Ampere ベース SKU の検討

#### 購入モデル最適化
| モデル | 割引率 | 期間 | 柔軟性 |
|-------|--------|------|--------|
| **On-Demand** | 0% | なし | 最高 |
| **1年 Reserved** | 〜30% | 1年 | SKU/リージョン固定 |
| **3年 Reserved** | 〜50% | 3年 | SKU/リージョン固定 |
| **Savings Plan** | 〜25% | 1-3年 | Compute ファミリ柔軟 |
| **Spot VM** | 〜90% | なし | いつでも回収される |
| **Dev/Test** | 〜50% | EA/MCA | 開発環境のみ |

#### アーキテクチャ最適化
- **サーバーレス活用**: 低トラフィック時のコスト削減
- **自動スケーリング**: 需要に応じた柔軟なスケーリング
- **アクセス層最適化**: Blob Storage の Hot/Cool/Archive 活用
- **データ転送最適化**: リージョン内通信の最大化、CDN 活用

### 3. FinOps プラクティス

| フェーズ | アクション |
|---------|----------|
| **Inform** | コスト可視化、タグベースの配賦、Cost Management レポート |
| **Optimize** | 未使用リソース削除、右サイズ化、Reserved/Savings Plan |
| **Operate** | 予算アラート、Anomaly Detection、定期レビュー |

### 4. 予算アラート設計

```
Budget: $X,XXX/月
├── 50% 消化 → 通知（Info）
├── 75% 消化 → 通知（Warning）
├── 90% 消化 → 通知（Critical）+ Action Group
├── 100% 消化 → 通知（Critical）+ 自動スケールダウン（オプション）
└── 110% 超過 → 通知（Emergency）+ 承認フロー
```
